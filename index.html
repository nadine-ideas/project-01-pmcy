1. SISTEMAS CENTRAIS E FUNIS (RD CRM)
Objetivo Principal: Garantir a escalabilidade e segregação das operações de vendas (funis e equipes) dentro do RD CRM para as novas diretorias (DICOM II e DICOM IV). [Calls]
Escopo: Escalar a implementação para DICOM II & DICOM IV. [KickOff]
Necessidade: Criar novos funis e segregação de equipes (e.g., "Vendas Transporte DICOM II"). [Bloco Notas]
Desafio Atual: Integração olha apenas o sufixo "Vendas Transportes", não segmenta por filial/diretoria. [Calls]
2. INTEGRAÇÕES CHAVE (Sistemas Conectados)
Objetivo Principal: Automatizar a comunicação de dados entre o RD CRM e sistemas críticos (CRM Web e Quiver), eliminando a necessidade de processos manuais de DE/PARA. [Levant. Reqs.]
CRM Web (Interno): Mapeamento automático do campo Responsável (Hierarquia), eliminando DE/PARA manual. [Doc API Cadastro]
Quiver (Externo): Automatizar DE/PARA de Campanha e Motivo de Perda (Funil Caminhão - GEAVE). [Doc API Leads]
SIGS (Interno): Comunicação direta (Cotação/Proposta) é uma Ideia de Futuro. [Levant. Reqs.]
3. CAMADA DE INTEGRAÇÃO (iPaaS/Middleware)
Objetivo Principal: Formalizar e implementar uma camada de intermediação (iPaaS/Middleware) para centralizar a lógica de transformação de dados (DE/PARA) e garantir a segurança das APIs. [Calls]
Mecanismo: APIs (Padrão REST) - Até 3 APIs na Fase 3. [Docs APIs]
iPaaS: Plataforma de Integração (Previsão: Fase 3 do Onboarding). [KickOff]
Função Central: Mapeamento e Transformação de Dados (DE/PARA). [Arquitetura]
Segurança: Autenticação via MAAS e Keycloak (Bearer/Basic). [Arquitetura]
4. MAPEAMENTO DE DADOS (DE/PARA) E REGRAS
Objetivo Principal: Substituir o trabalho manual de ETL por automação, implementando regras de DE/PARA que considerem as novas estruturas (Diretoria/Filial) e requisitos de campo (Customizados). [Bloco Notas]
Regra Hierarquia: Incluir campos Diretoria e Filial no DE/PARA (trds_depara_crm_hierarquia). [DexPara-Hier.]
Regra Campanha/Motivo de Perda: Permitir cadastro direto no RD (trds_depara_crm_campanha/mot_perda). [DexPara-Camp.] [DexPara-Mot.P]
Campos Personalizados: Cruciais para integração (ex: endereço). Devem manter o formato do CRM Web e NÃO podem usar autofill (Google Maps). [Calls/Requisitos]
